# CCP Course Structure

## Official Hierarchy

Course → Level → Lesson → Session → Active Blocks → Concept Theory → Demonstrations → Pseudocode → Programming Commands → Implementations → Exercises → Debugging → Correction → Mistake Log → Command Gap Log → Git Log → Grade Log → Archive

## Master Size

6 Levels × 8 Lessons × 4 Sessions = 192 Sessions

Each lesson has 4 sessions:

1. Concept + Pseudocode + First Implementation
2. Guided Practice + Second Implementation + Debugging
3. Mixed Programming + CLI + Application / Research Link
4. Review + Mini-Assessment + Correction + Git Archive

## Exercise System

Each session assigns exactly 13 exercises:

- 4 Mandatory Core
- 4 Recommended
- 3 Stretch
- 2 Research/Application Integration

## Difficulty Ladder

THE Easiest → Easiest → Easier → Very Easy → Easy → Pre-Moderate → Moderate → Post-Moderate → Hard → Very Hard → Harder → Hardest → THE Hardest

## Exercise Types

- D — Drill / Trace
- L — Learning / Explanation
- C — Command / Syntax
- A — Application / Project

## Session Parts

### Part 0
Energy/Fatigue check and mode selection.

### Part 1
Theory, examples, pseudocode, syntax table, implementation target, CLI/Git operation, debugging notes, research connection.

### Part 2
Exactly 13 exercises. No solutions before attempt.

### Part 3
Correction, logs, grades, archive, next session preview.

## Logs

- Mistake Log
- Command Gap Log
- Prerequisite Gap Log
- Git Log
- Debugging Log
- Research Readiness Log
- Track Maturity Update
- Hardest/THE Hardest Readiness Update
